    DOM & Pipeline Concepts

    1. Browser Rendering Pipeline
    • HTML Parsing & Tokenization: The browser reads raw bytes, converts them to characters, tokens, nodes, and finally builds the DOM tree.
    • CSSOM Tree Creation: CSS rules are parsed into the CSS Object Model tree.
    • Render Tree: The visual combining of DOM and CSSOM. Nodes that are hidden (`display: none`) are omitted.
    • Layout & Painting: The browser computes positions/sizes (Layout) and draws pixels on the screen (Painting).
    2. DOM Tree & Manipulation

    The DOM (Document Object Model) is a structural tree of objects representing HTML. Methods like append(), prepend(), after(), before(), replaceWith(), and remove() let us dynamically alter this architecture in JS.

    3. Attributes vs Properties

    Attributes are defined by HTML tags and are generally static strings representing default state.
    Properties are values belonging to the DOM object node in Javascript. They represent current live state and can be booleans, arrays, objects, etc.

    4. Event Bubbling & Capturing

    Bubbling: The event triggers on the deepest target element, then triggers upwards (Child -> Parent -> Grandparent).
    Capturing: The event travels downwards from the root to the target element (Grandparent -> Parent -> Child).

    5. Event Delegation

    Instead of manually attaching event listeners to each task card, we place one single addEventListener on the parent container. We then check event.target to figure out what was clicked. This drastically improves memory efficiency and automatically handles future dynamic elements.
